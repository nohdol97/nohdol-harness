# 제안: 내부 통신 케이브어(전보체) 전환 검토 — 기각

- 날짜: 2026-07-22
- 상태: 기각 — §15 영어 규칙 유지, 전면 전보체 전환 안 함 (재검토 반복 방지용 근거 기록)
- 출처: 사용자 검토 요청("케이브어로 적용하는 거 검토해줘 — 토큰 더 줄지 않아?")

## 검토 대상

§15 내부 통신(발행 프롬프트·`_workspace/` 중간 리포트·P2P·반환 텍스트)을 영어에서 **케이브어(caveman speak/전보체 — 관사·전치사·접속사를 깎은 압축 영어)**로 바꿔 토큰을 추가 절감하는 안.

## 기각 근거

1. **절감이 작다**: §15 한국어→영어는 같은 내용·같은 충실도의 무손실 인코딩 전환(~1.5-2배)이었지만, 전보체는 기능어를 깎는 **손실 압축**이다. 영어 기능어는 BPE에서 대부분 1토큰 이하라 산문 절감이 15~25%에 그치고, 내부 산출물의 상당 부분(코드·경로·`파일:줄`·JSON·표)은 애초에 압축 여지가 없다 — 전체 실효 절감 추정 10~15%.
2. **리스크가 절감을 압도한다**: 기능어가 범위·부정·조건("unless"·"only if"·"not A but B")을 나른다. 발행 프롬프트의 모호함 1건 = 서브에이전트 재발행 1회(실측 리뷰 1회 ~50k 토큰)인데, 프롬프트 2천 토큰의 15% 절감은 300토큰 — 비대칭이 극단적. 검증·판정 텍스트(must-fix 근거·PASS/BLOCK)는 정밀성이 생명이고, §16의 원칙("짧은 코드가 빠뜨린 코드의 알리바이가 되면 안 된다")의 언어판에 해당한다. 모델의 문체 미러링으로 전보체 리포트가 재생산되면 N회 재독 채널(integrator·메인 루프)의 오독 확률이 함께 오른다.
3. **깎아도 안전한 곳은 이미 깎여 있다**: 리포트 요약 인덱스 줄(ID·severity·요지), team-log 이벤트 서술, 발행 label은 사실상 전보체가 정착돼 있다. 남은 산문은 깎으면 위험한 곳이라 명문화 가치도 낮다(규칙 자체가 로딩 토큰 — §8 ④).

## 대신 채택된 절감 축 (같은 날 별도 적용 — 이 제안의 대체재)

입력 토큰 절감은 손실 압축이 아니라 **적재 구조**에서 얻는다: CLAUDE.md 앵커 다이어트(7,024→6,328B), 리뷰 diff 10KB 인라인 임계, integrity-check R14·R15(기계 이관), 리뷰 검증 범위 상한, 팀원 공통 읽기 규율 ⑧, rtk(명령 출력 인제스천) — changelog 2026-07-22 참조.

## 재검토 조건

토크나이저·모델 세대 변화로 "전보체 지시의 준수 정밀도가 자연문과 동등하다"는 실측이 나오면 발행 프롬프트 한정으로 재검토할 수 있다 — 그 전까지는 닫는다.
