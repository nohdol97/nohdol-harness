# ADR 016 — 내부 통신 언어 정책 (모델만 읽으면 영어, 사용자가 읽으면 한국어)

- **날짜**: 2026-07-15
- **변경 내용**: 15절 신설 — 서브에이전트 발행 프롬프트, `_workspace/` 팀 중간 리포트, P2P 메시지 본문은 **영어**로 쓴다. 사용자 대면 출력·하네스 문서·사용자에게 경로가 전달되는 판정 문서(integrator 최종 리포트, 런북, 제안서, 운영 로그, 대기 큐)·한국어 트리거 키워드는 **한국어 유지**. 언어 섞임 방지 가드 3항(소화 보고, 원문 인용 원어 유지, 애매하면 한국어) 포함.
- **대상**: AGENTS.md 15절, 에이전트 정의 7종(3절 출력 언어), orchestrate(공통 팀원 규칙·최종 응답), team-review, doc-writer(문체 규칙·작업 리포트 템플릿), harness-review(실행 방식)
- **사유**: 사용자 질문(2026-07-15) "내부 통신을 영어로만 하면 토큰 절약 성과가 있을까" + 부작용 없는 적용 요청. 현대 토크나이저에서 한국어는 같은 내용 기준 영어의 약 1.5~2배 토큰이고, 팀 중간 산출물은 쓰기 1회 + 읽기 N회(integrator·메인 루프 재독)라 절감이 곱으로 붙는다. 스킬 description 영문 규칙(metaskill 공통 규칙 2)이 같은 원리의 선례.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 기준은 읽는 주체 | 모델만 읽는 산출물 → 영어, 사용자가 읽는 산출물 → 한국어 | "내부/외부" 같은 위치 기준은 경계 사례(최종 리포트, 런북)에서 갈린다 — 독자 기준은 모든 사례에서 판정 가능하다 |
| 판정 문서는 한국어 | integrator 최종 리포트·배포 런북·제안서·운영 로그·대기 큐는 `_workspace/`에 있어도 한국어 | 사용자에게 경로를 전달해 직접 읽도록 안내하는 문서다 — 절감보다 소통 실패(승인 대상을 못 읽음) 비용이 크다 |
| 섞임 방지 가드 명문화 | 영어 산출물은 한국어로 소화해 보고, 최종 보고 직전 언어 확인, 애매하면 한국어 | 영어 컨텍스트가 길수록 한국어 출력이 번역투·언어 섞임으로 미끄러진다 — 사용자 요청("부작용 발생하지 않게")의 직접 대응 |
| 하네스 문서는 한국어 유지 | AGENTS.md·SKILL.md 등 항상 로드 문서의 영어화는 하지 않는다 | 매 세션 로딩 고정비 절감은 크지만, 사용자가 직접 읽고 유지보수하는 운영 자산이다 — 내부 통신과 별개 결정으로 분리(이번 범위 밖) |

## 기각 대안

| 대안 | 기각 사유 |
|---|---|
| 전면 영어화(하네스 문서 포함) | 최대 절감이지만 하네스의 1차 독자는 사용자 — 가독성·유지보수성 훼손이 절감을 상회 |
| 현상 유지(전부 한국어) | 팀 팬아웃 작업의 중간 리포트가 토큰 소모의 큰 몫인데 전부 모델만 읽는다 — 근거 있는 절감 기회 방치(④ 수축·효율) |
| 자동 언어 감지·번역 계층 | 강제 장치가 없는 데다 번역 자체가 토큰을 쓴다 — 규칙 + 에이전트 정의 반영이 더 싸고 확실 |

## 검증

효과 실측은 주간 점검(④ 신호)의 agentsview `stats`로 전후 비교한다 — 적용 전 주간 대비 팀 작업 세션의 토큰 소모 추이.
