# ADR 006 — 하위 프로젝트 하네스 중앙 관리 (원본 루트 일원화 · 미추적 · 배포 장치 없음)

- **날짜**: 2026-07-13
- **변경 내용**: 하위 프로젝트 하네스의 원본이자 유일본을 루트 `.agents/projects/<이름>/`에 두고, REGISTRY.md처럼 설치처별 데이터로 **git 미추적**한다. `project/<이름>/`와 그 git 저장소에는 하네스 파일을 일절 두지 않는다(심링크·복사본 포함) — 라우팅(REGISTRY.md → 원본 읽기)이 연결한다. 하위 전용 스킬·에이전트는 루트 `.agents/`에 `<프로젝트>-` 접두어로, 하위 전용 훅은 루트 settings에 경로 분기형으로만 둔다.
- **대상**: AGENTS.md 5·12절, .gitignore, `.agents/projects/`(신설), metaskill SKILL.md·scaffold.md, orchestrate SKILL.md(공통 팀원 규칙 ①), harness-review·harness-install SKILL.md, README
- **사유**: 사용자 확정(2026-07-13) — 모든 하위 하네스 관리는 루트 워크스페이스에서 하고, 컴퓨터마다 두는 프로젝트가 다르므로 미추적한다(ADR 005 이식성 원칙의 연장). 같은 날 검토된 "심링크 배포 + `.git/info/exclude`" 안은 REGISTRY.md 라우팅이 이미 원본 읽기를 강제해 배포 장치가 중복이라는 사용자 판단으로 폐기했다.

## 배경 — cwd 로드 문제

Claude Code의 스킬·훅·권한 설정은 세션 시작 디렉토리(cwd) 기준으로만 로드된다. 하위 프로젝트에 실행 계층을 분산하면 루트 세션(크로스 프로젝트)에서 로드되지 않고, 그 역도 마찬가지다. 문서(AGENTS.md)는 라우팅 규칙이 읽기를 강제하므로 cwd와 무관하게 작동하는 유일한 계층이다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 원본 일원화 | 하위 하네스는 `.agents/projects/<이름>/`(AGENTS.md·CLAUDE.md·adr/)이 유일본. 배포(심링크·복사) 없음 | 배포본은 원본과 어긋나는 순간부터 부채. 라우팅이 원본 읽기를 강제하므로 중복 장치 불필요 |
| 미추적 | `.agents/projects/`는 gitignore (README만 추적 예외로 디렉토리 유지) | 컴퓨터마다 프로젝트가 다름 — REGISTRY.md와 동일한 설치처별 데이터(ADR 005) |
| 트레이드오프 수용 | 하위 하네스는 어느 git에도 없음 → 이력·원격 복구 불가 | 사용자 수용(2026-07-13). 대신 하위 프로젝트 저장소가 언제나 깨끗하고, 설치처 간 충돌이 원천 차단됨 |
| 라우팅이 연결 | REGISTRY.md에서 프로젝트 식별 → `.agents/projects/<이름>/AGENTS.md` 읽기 (7절 2단계의 실제 경로). `<이름>`은 레지스트리 행과 일치 | 레지스트리가 이미 라우팅의 단일 근거 — 별도 연결 장치를 만들면 원본이 둘이 됨 |
| 규칙은 문서 우선 | 가드레일·필수 규칙은 하위 AGENTS.md에만 싣고, 스킬·훅은 편의 장치로 취급 | cwd 로드 문제 — 문서만이 어디서든 작동 |
| 스킬·에이전트 루트 단일 소스 | 하위 전용도 루트 `.agents/`에 `<프로젝트>-` 접두어로 생성(추적됨) | 하위에 두면 루트 세션에서 안 보임. 프로젝트 없는 설치처에서는 트리거되지 않을 뿐 무해 |
| 훅은 루트 경로 분기형 | 루트 `.claude/settings.json`에서 스크립트가 대상 경로 검사 | 훅은 세션 루트 것만 실행 — 우회로 없음 |
| 세션은 루트에서 | 하위 프로젝트 작업 포함 전부 루트에서 세션 시작 | `project/<이름>/`에서 열면 하네스가 전혀 로드되지 않음 |
| 팀원 하네스 주입 | orchestrate 팀원·서브에이전트는 작업 전 `.agents/projects/<이름>/AGENTS.md` 읽기 규칙화 | 서브에이전트도 하위 훅·설정을 상속받지 못함 |
