# ADR 030 — 하네스 모델-로드 자산의 영어 단일 원본 전환

- **날짜**: 2026-07-22
- **상태**: 채택
- **대상**: AGENTS.md, CLAUDE.md, `.agents/agents/*.md`(7종), `.agents/skills/*/`(SKILL.md+references 16종), metaskill 산출 언어 규칙(§15 개정), 신규 한글 뷰 3종, integrity-check 드리프트 가드

## 변경 내용

1. **영어 단일 원본**: 모델이 세션·발행마다 로드하는 하네스 자산 — AGENTS.md, CLAUDE.md, 에이전트 정의 전부, 스킬 전부(SKILL.md·references) — 를 영어로 전환한다. 이후 metaskill이 생성하는 하네스 자산도 기본 영어로 작성한다.
2. **한글 뷰(사용자 이해도 계층)**: ① 루트 `AGENTS.ko.md` — AGENTS.md 규칙의 한글 다이제스트(생성된 뷰, 편집 금지 배너 + source-hash), ② `.agents/agents/README.ko.md` — 에이전트 전체 한글 요약, ③ `.agents/skills/README.ko.md` — 스킬 전체 한글 요약. 원본 변경 시 같은 커밋에서 재생성한다.
3. **한글 유지**: 사용자 대면 출력 전부(채팅·PR·커밋·이슈 — §15 불변), ADR·스펙·changelog·루트 README(사용자 이력 자산), 스킬 description 내 한국어 재실행 키워드(사용자 입력 매칭 기능), 사용자 대면 문서 템플릿(doc-writer 스펙·PR 본문·런북 등).
4. **드리프트 가드(기계 점검)**: `integrity-check.py`에 신규 규칙 — ⓐ AGENTS.ko.md 배너의 source-hash가 현재 AGENTS.md 해시와 일치, ⓑ 한글 요약 2종의 항목 집합이 실제 `.agents/agents/`·`.agents/skills/` 집합과 1:1. 뷰 재생성 누락을 커밋 전에 잡는다.
5. **§15 개정**: 판단 기준 "읽는 주체"는 유지하되, 하네스 운영 자산을 모델-read로 재분류한다(아래 근거).

## 사유

- **읽기 빈도 실측**: AGENTS.md는 매 세션 양 CLI에 주입되고(한글 9,313자 ≈ 세션당 ~5–9k 토큰 초과 지출), hot 스킬·에이전트 본문은 작업 세션마다 재로드된다(한글 46,539자). §15의 원칙 "같은 내용 기준 한국어는 영어보다 1.5~2배 토큰, 쓰기 1회·읽기 N회라 절감이 곱으로 붙는다"를 읽기 빈도에 적용하면 이 자산들은 압도적으로 모델-read다 — 사람이 읽는 빈도는 그에 비해 미미하다.
- **예산 포화**: 2026-07-22 주간 점검 실측 — AGENTS.md 39,930B로 R14 예산 40,000B 대비 여유 70B. 규칙 하나만 추가해도 기계 점검이 터지는 상태였고, 영어 전환은 산문 다이어트(P3, ~1.6KB)보다 한 자릿수 큰 여유를 만든다.
- **사용자 이해도**: 전문 병렬 사본(수기 이중 원본)은 드리프트 부채라 기각(§12 "배포본은 원본과 어긋나는 순간 부채", ADR 027 얇은 어댑터 전례). 대신 생성된 한글 뷰 + 기계 드리프트 가드로 이해도와 단일 원본을 양립시킨다(2026-07-22 사용자 선택: 생성 뷰 A안 → 같은 날 요약 뷰로 최종 확정).

## 트레이드오프 (수용)

- 번역 시점 의미 드리프트 위험 → reviewer 독립 검증(규칙 1:1 대조, §3·§13 정밀)으로 완화. 전 과정 git 복구 가능.
- 한글 뷰는 요약이라 세부 규칙 판단은 영어 원본 참조 필요 → 뷰 상단 배너에 명시.
- ADR·스펙·changelog는 한글 유지로 온디맨드 로드 시 토큰 비효율 잔존 → 로드 빈도가 낮고 사용자 이력 자산이라 수용(티어 3 분석, 2026-07-22).
