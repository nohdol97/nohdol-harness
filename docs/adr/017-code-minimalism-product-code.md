# ADR 017 — 코드 최소주의(제품 코드) 채택 — ponytail 결정 사다리 이식

- **날짜**: 2026-07-16
- **변경 내용**: AGENTS.md 16절 신설 — 하위 프로젝트 제품 코드 작업에 ponytail의 **결정 사다리 7단계**(필요성→기존 코드→표준 라이브러리→네이티브→기존 의존성→한 줄→최소 구현)와 "문제 이해 뒤에 오른다"·"동작하는 가장 짧은 diff" 규율, 예외 영역(엄밀성 유지)을 이식. team-review 관점 축에 "단순성/과설계"(태그 5종·`net: -N lines` 출력) 추가. **원칙 + 리뷰 관점 1개만 이식**하고 ponytail의 배송 기계(Claude Code 플러그인·Node.js 훅·6스킬)는 채택하지 않음.
- **대상**: AGENTS.md 16절·변경 이력, `.agents/skills/team-review/SKILL.md`(관점 표), `docs/proposals/2026-07-15-ponytail-adoption.md`(설계·비목표)
- **사유**: 사용자 요청(2026-07-16) — "ponytail(github.com/DietrichGebert/ponytail, MIT)을 분석하고 이 하네스에 적용" → 설계 제안 검토 후 "권장(최소 채택) 적용". 이 하네스의 최소주의는 **하네스 자산**(ADR 007 지연 생성, 신호 ④ 수축·효율)에 한정돼 있고, 에이전트가 하위 프로젝트에 쓰는 **제품 코드 분량**을 규율하는 축이 공백이었다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 제품 코드 최소주의를 AGENTS.md에 신설 | 16절에 결정 사다리·예외 영역을 문서로 이식, 적용 범위=하위 프로젝트 제품 코드 | 규칙의 유일한 운반체는 문서다(12절). 훅·플러그인은 cwd 로드 편의 장치라 원본이 될 수 없다 |
| ADR 007과 별개 축으로 명시 | 지연 생성=하네스 비대 방지, 16절=제품 코드 분량 규율 | 두 축을 섞으면 "하네스도 최소로"와 "제품 코드도 최소로"가 혼선된다 |
| 예외 영역은 완화 불가로 못박음 | 검증·에러 처리·보안·접근성·명시 요청 기능, 특히 3절 가드레일·13절 완료 기준 | "짧은 코드"가 "빠뜨린 코드"의 알리바이가 되면 최소주의가 무모함으로 변질된다 |
| 리뷰는 새 스킬 없이 team-review 관점 축으로 | 기존 관점 팬아웃에 "단순성/과설계" 1행 추가(태그 5종·삭제 목록·`net: -N`) | 새 스킬은 ADR 007(지연 생성) 위반이자 CLAUDE.md 리뷰→team-review 라우팅 우회. 사다리 2번(재사용)의 실천 |
| 제품 shrink와 하네스 shrink 분리 | 제품 코드=team-review, 하네스 자산=harness-review ④ | harness-review는 하네스만 본다 — 대상이 다르면 도구도 다르다 |

## 기각 대안

| 대안 | 기각 사유 |
|---|---|
| Claude Code 플러그인 설치(`/plugin install ponytail`) | 11절: 외부 도구는 전역 설치·보조 도구로만. 플러그인 훅은 cwd 로드라 심링크 규율·Codex 비호환과 충돌 |
| Node.js 훅 이식(mode-tracker·subagent 주입·statusline) | 이 하네스 훅은 Python `_common.py` 기반 — Node 런타임 추가는 사다리 5번 위반. 서브에이전트 무차별 룰 주입은 스코프 도구·티어·영어 발행 프롬프트(10·15절)를 오염 |
| ponytail 6스킬 전부 이식 | 대부분 기존과 중복(review→team-review, debt→work-tracker, audit→team-review 팀 모드). 지연 생성 원칙 위반 |
| `/ponytail-gain` 벤치마크 채택 | 벤치마크 근거가 n=4·Haiku 4.5·단일 저장소라 표제 수치를 명분으로 과신 불가 — 유지 비용 대비 가치 낮음 |

## 검증

- **dogfooding**: 16절 신설 후 team-review "단순성/과설계" 축이 실제 diff에서 삭제 목록을 산출하는지 1회 확인(제품 코드 대상 작업 발생 시).
- **효과 실측**: 제품 코드 분량 감소는 프로젝트별 PR 리뷰에서 관찰 — 이 하네스 저장소는 문서 중심이라 자체 측정 대상이 아니다.

## 라이선스

ponytail은 MIT. 결정 사다리·예외 영역 문구를 이식하므로 16절 하단과 이 ADR에 출처·라이선스를 명기했다(MIT는 저작권 고지 유지가 조건).
