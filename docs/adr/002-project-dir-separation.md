# ADR 002 — 하위 프로젝트 저장소 분리 (`project/`, `dev/` 미추적)

- **날짜**: 2026-07-12
- **변경 내용**: 하위 프로젝트를 `project/<이름>/`에 두되 루트 저장소에서 gitignore 처리. 실험·임시 개발 공간 `dev/`도 미추적. README.md 신설, AGENTS.md 1절(경로 규약)·5절(저장소 분리) 갱신, scaffold.md 절차 수정.
- **대상**: .gitignore, AGENTS.md, README.md, metaskill/references/scaffold.md
- **사유**: 사용자 확정(2026-07-12) — Claude Code / Codex는 이 루트에서 열어 작업을 지시하지만, 프로젝트들은 각자 독립 git 저장소로 관리한다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 저장소 분리 | 루트 저장소는 하네스만 추적, `project/<이름>/`은 각자 독립 저장소 | 프로젝트마다 배포·CI 주기가 다름. 하네스 이력이 프로젝트 커밋에 묻히지 않게 함 |
| `dev/` 미추적 | 실험·임시 개발 공간, 레지스트리 등록 대상 아님 | 세션성 산출물로 하네스도 프로젝트도 아님 |
| 하위 하네스 위치 | 하위 프로젝트의 AGENTS.md 등 하네스 파일은 **해당 프로젝트 저장소에** 커밋 | 루트가 project/를 추적하지 않으므로 하네스를 잃지 않으려면 프로젝트 저장소에 있어야 함 |
| 레지스트리 유지 | 루트 AGENTS.md 레지스트리가 계속 라우팅의 단일 근거 (경로만 `project/` 접두) | CLI를 루트에서 열어 지시하는 운영 방식은 변하지 않음 |

## 영향

- ADR 001의 "모노 워크스페이스 단일 저장소" 전제가 "하네스 루트 + 독립 프로젝트 저장소들"로 대체된다.
- metaskill의 신규 프로젝트 스캐폴딩은 `project/<이름>/`에 `git init`을 포함한다 (scaffold.md 갱신 완료).
