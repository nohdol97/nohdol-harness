# ADR 024 — `dev/` 실험 공간 디렉토리 제거

- **날짜**: 2026-07-19
- **변경 내용**: 루트 워크스페이스의 `dev/`(실험·임시 개발 공간, ADR 002에서 도입)를 제거한다. 모든 실작업은 `project/<이름>/`에 둔다. 딸린 변경: `.gitignore`에서 `dev/` 항목 삭제, AGENTS.md 1·3·5·13절의 `dev/`·"실험 공간" 언급 삭제, README.md 디렉토리 트리·설치 안내 정정, harness-install SKILL(디렉토리 생성·인터뷰·체크리스트)에서 `dev/` 삭제, REGISTRY.md 경로 규약 정정, **tdd-gate 훅의 `dev/` 예외 분기 제거**(및 회귀 테스트 C6b 삭제·스펙 R2 정정), secret-gate 주석 정정.
- **대상**: .gitignore, AGENTS.md, README.md, REGISTRY.md, `.agents/skills/harness-install/SKILL.md`, `.agents/githooks/tdd-gate.py`·`tdd-gate_test.py`·`secret-gate.py`, `docs/specs/2026-07-13-tdd-gate-hook.md`·`docs/specs/2026-07-18-secret-gate-hook.md`, ADR 002·008(개정 배너)
- **사유**: 사용자 확정(2026-07-19) — "dev는 없애자. project에만 작업할 것들 넣는걸로." 실측상 `dev/`는 비어 있었고(사용 0), 두 개의 미추적 최상위 공간은 경로 규약을 갈래지게 하고 문서·훅 여러 곳에 예외를 유지하는 비용만 남겼다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| `dev/` 제거 | 실험 공간을 별도 최상위 디렉토리로 두지 않는다. 실작업은 전부 `project/<이름>/` | 미사용(빈 디렉토리)이었고, 별도 공간은 라우팅·경로 규약·훅 예외의 유지 비용만 발생 |
| TDD 예외 축소 | tdd-gate의 제외 저장소(R2)에서 `dev/`를 뺀다 — 남는 예외는 **루트 하네스 저장소(문서 중심)** 하나 | "project에만 작업"은 곧 모든 실작업이 완전한 SDD+TDD 규율을 받는다는 뜻. 실험용 TDD 우회의 근거가 사라짐 |
| 우회 경로 유지 | 동작 불변 수정의 `[no-test]`(사용자 확인 전제)·재조합 커밋·최초 커밋 예외는 그대로 | 이들은 실험 공간과 무관한 정당한 예외이며, 커밋 메시지에 흔적이 남거나 게이트가 이미 지난 커밋의 재조합이다 |

## 대체 관계

ADR 002가 도입한 `project/`·`dev/` 미추적 결정 중 **`dev/` 부분만 이 ADR로 대체**한다(`project/` 분리는 유효). ADR 008이 명시한 TDD 게이트 예외 경로 목록에서 **`dev/` 항목이 이 ADR로 제거**된다(나머지 예외·차단 지점·fail-open 결정은 유효). 두 ADR 상단에 부분 개정 배너를 단다.

## 비고

`dev/` 제거는 디스크의 빈 디렉토리를 지우는 것을 포함하지 않아도 무해하다(미추적이라 git에 영향 없음). 향후 일회성 실험이 필요하면 `project/` 밖 임의 경로나 스크래치패드를 쓰고, 저장소로 승격할 때 `project/`에 편입한다.
