# ADR 007 — 하위 스킬·에이전트 지연 생성 + 프로젝트 로컬 배치

- **날짜**: 2026-07-13
- **변경 내용**: ① 초기 하네스 키트(신규 하네스 생성 시 스킬·에이전트 선제 생성) 폐지 — 반복 작업 관찰 또는 작업 중 등록 가치 발견 시에만 사용자 승인으로 생성(지연 생성). ② 하위 전용 스킬·에이전트의 위치를 루트 `.agents/skills/<프로젝트>-*`(추적)에서 **`.agents/projects/<이름>/skills/`·`agents/`(미추적)**로 변경. ③ 하위 프로젝트 CLAUDE.md를 규격에서 삭제.
- **대상**: AGENTS.md 12절, metaskill SKILL.md(생성 시나리오·생성물 요건·체크리스트), scaffold.md 3·3.5절, `.agents/projects/README.md`
- **사유**: 사용자 확정(2026-07-13) — "초기에는 만들지 말고, 반복·등록 가치가 보일 때 프로젝트별로 만들되 git에 들어가지 않게."

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 지연 생성 | 신규 하네스의 생성물은 문서(AGENTS.md·adr/)뿐. 인터뷰의 반복 작업 예상은 하위 AGENTS.md "스킬 후보" 섹션에 기록만 | 미리 만든 스킬은 실사용과 어긋나 우회된다. 필요가 증명된 것만 만든다(진화 트리거와 동일 철학) |
| 프로젝트 로컬 배치 | 생성 위치는 `.agents/projects/<이름>/skills/`·`agents/` — `.agents/projects/` 미추적에 포함되어 git에 안 들어감 | 프로젝트 스킬은 설치처별 데이터다. 루트 접두어 방식(ADR 006)은 설치처별 스킬이 공용 저장소에 추적되는 모순이 있었다 |
| 로드는 문서 경유 | 이 위치는 CLI가 자동 발견하지 못함 — 하위 AGENTS.md에 목록·경로를 명시하고 라우팅이 읽게 한다 | "규칙의 운반체는 문서" 원칙과 동일 메커니즘. 자동 트리거 상실은 수용한 트레이드오프 |
| 승격 경로 | 여러 프로젝트·설치처에 일반화되는 스킬만 프로젝트 특화 내용을 제거해 루트 `.agents/skills/`로 승격(추적) | 공용 저장소에는 공용 규칙만 |
| 하위 CLAUDE.md 삭제 | 하위 하네스에 CLAUDE.md를 만들지 않음 | CLAUDE.md는 세션 시작 디렉토리 자동 로드용인데 `.agents/projects/`에서 세션을 시작할 일이 없다 — 로더 없는 죽은 파일 |
