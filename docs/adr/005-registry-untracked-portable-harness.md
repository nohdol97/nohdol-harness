# ADR 005 — 하네스 이식성: REGISTRY.md 미추적 전환, 경로 규약 이관, 티어 표 탈(脫)모델명

- **날짜**: 2026-07-12
- **변경 내용**: ① REGISTRY.md를 gitignore로 전환(추적 해제) ② 경로 규약을 AGENTS.md 1절에서 REGISTRY.md로 이관 ③ 티어→모델 매핑 표에서 구체 모델명(Opus/Sonnet/gpt-5-codex 등) 제거, CLI 무관 선택 기준으로 대체 ④ harness-install·project-status 스킬 신설.
- **대상**: .gitignore, AGENTS.md 1·5·9절, REGISTRY.md, .agents/skills/(2종 추가), 에이전트 3종 티어 문구
- **사유**: 사용자 확정(2026-07-12) — 이 하네스는 여러 컴퓨터에 설치해 쓴다. 설치처마다 다른 것(레지스트리, 경로 규약)은 저장소에 넣으면 안 되고, 시간이 지나면 바뀌는 것(모델명)은 하네스에 고정하면 안 된다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| REGISTRY.md 미추적 | gitignore 등재 + `git rm --cached`. ADR 004의 "REGISTRY.md 추적" 결정을 대체 | 컴퓨터 A의 프로젝트 목록이 컴퓨터 B에 클론되면 B의 라우팅이 처음부터 오염된다 |
| 설치 시 생성 의무 | 새 설치처에서는 REGISTRY.md를 가장 먼저 생성 — harness-install 스킬이 절차(심링크 검증→디렉토리 생성→스캔→인터뷰→생성) 담당. REGISTRY.md 부재 = 설치 미완료 상태로 간주 | 생성 의무를 명시하지 않으면 첫 세션이 라우팅 근거 없이 작업을 시작한다 |
| 경로 규약 이관 | `project/<이름>/`·`dev/` 규약을 AGENTS.md(공용)에서 REGISTRY.md(설치처별)로 이동 | 프로젝트를 어디에 두는지도 설치처의 선택이다. 공용 문서에는 "독립 git 저장소" 원칙(ADR 002)만 남김 |
| 티어 표 탈모델명 | "design=Opus, Codex=gpt-5-codex(high)" 식 고정 대신 "해당 CLI의 최고 성능 모델 / 표준 모델" 선택 기준으로 서술. Haiku 금지도 "경량(최저가) 모델 금지"로 일반화 | 모델명은 분기마다 바뀐다. 이름을 박제하면 하네스가 낡는 속도가 모델 릴리스 속도와 같아진다 |
| 스킬 2종 신설 | harness-install(설치 부트스트랩), project-status(전체 현황 팬아웃 리포트) | 전자는 미추적 전환의 필연적 짝. 후자는 멀티 프로젝트 관리 하네스의 최빈 질문("전체가 어떤 상태인가")의 고정 절차화 |
| 에이전트는 미추가 | 표준 로스터 4종 유지 | 범용 역할(수집·구현·검증·통합)은 이미 커버. 도메인 특화는 하위 프로젝트 하네스의 몫(ADR 004) |
