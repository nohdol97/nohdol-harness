# ADR 021 — CLAUDE.md의 `@AGENTS.md` 임포트 (단일 원본 항상-온) + 변경 이력 분리

- **날짜**: 2026-07-16
- **변경 내용**: CLAUDE.md가 AGENTS.md를 **산문 링크**로 가리키던 것을 **`@AGENTS.md` 임포트**로 바꿔, 단일 원본 전문이 세션 시작 시 자동 주입(항상-온)되게 한다. 임포트 footprint를 상쇄하려 **변경 이력 테이블을 `docs/harness-changelog.md`로 분리**(항상-온 대상에서 제외)하고, CLAUDE.md의 **중복 트리거(continuation·k8s 상세)를 슬림**해 Claude 전용 항상-온 앵커(출력 언어·라우팅 요약·스킬 우선순위)만 남긴다. 출력 언어 규칙(§15)을 CLAUDE.md 첫머리 항상-온 앵커로 승격한다.
- **대상**: `CLAUDE.md`(재작성), `AGENTS.md`(5·6·11·15절), `docs/harness-changelog.md`(신설), `docs/README.md`(MOC)
- **사유**: 사용자 관찰(2026-07-16) — 다중 에이전트 작업 후 **사용자 대면 요약이 영어로 새는** 문제. §15에 규칙(+가드)이 이미 있었지만 **AGENTS.md에만** 있어 항상-온이 아니었다. Claude Code는 CLAUDE.md만 자동 로드하고 AGENTS.md는 산문 링크로는 주입하지 않는다(공식 문서 확인 — `@경로` 임포트라야 주입, 마크다운 링크·산문은 모델이 `Read`할 때만). 서브에이전트 영어 리포트가 컨텍스트에 신선하게 남은 상태에서 최종 요약이 영어로 미끄러진 것. 근본 해결은 단일 원본을 항상-온으로 만드는 것.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| `@AGENTS.md` 임포트 채택 | CLAUDE.md 첫 줄 `@AGENTS.md`로 전문 자동 주입 | Claude Code 공식 문서가 **AGENTS.md 저장소의 권장 패턴**으로 명시(`@AGENTS.md` in CLAUDE.md). §15를 포함한 모든 항상-온 규칙의 누수를 근본 차단. Codex는 AGENTS.md를 네이티브 로드하므로 이 변경으로 **두 CLI 패리티**가 맞는다(누수는 Claude Code 특유였다) |
| 변경 이력 분리 | 41행 변경 이력 표를 `docs/harness-changelog.md`로 이동, AGENTS.md엔 포인터만 | 임포트는 토큰을 아껴주지 않는다(공식 문서 — 전문이 매 세션 로드). 감사 로그는 매 세션 필요 없으므로 임포트 대상에서 빼 footprint를 20~30% 줄인다. 이 하네스의 토큰 효율 철학(§15/ADR 016)과 정합 |
| CLAUDE.md 중복 트리거 슬림 | AGENTS.md §7과 중복되는 continuation·k8s 상세를 CLAUDE.md에서 축약, 스킬 맵·우선순위·출력 언어 앵커만 유지 | AGENTS.md가 항상-온이 된 이상 §7 상세를 CLAUDE.md가 복사할 이유가 없다. 앵커는 매 턴 salience용 요약으로만 — 상세는 §7 단일 원본 |
| 출력 언어 항상-온 앵커화 | §15 user-facing 규칙을 CLAUDE.md 첫머리 앵커로 승격(§15 원본은 유지) | `@import`(항상-온)와 앵커(요약, 최상단)의 **이중 보장** — 긴 문서 속 개별 규칙의 주목도 희석을 앵커가 방어 |
| CLAUDE.md 로딩 역할 규율 신설(§11) | "CLAUDE.md = `@AGENTS.md` + Claude 전용 항상-온 앵커만, 전문·이력 복사 금지" | 사용자 관찰("CLAUDE.md에 따로 든 정보가 많다") — 재비대 방지 규율을 명문화해 앞으로 CLAUDE.md가 제2 원본으로 갈라지지 않게 함 |
| 산문 링크 방식 폐기 | `[AGENTS.md](AGENTS.md) 읽어라`만으로는 부족 | 산문·마크다운 링크는 자동 주입 트리거가 아니다(공식 문서). 모델이 실제 `Read`할 때만 들어오고 compact 후 빠진다 — 누수의 근본 원인 |

## 트레이드오프 (정직한 평가)

- **비용**: `@import`는 AGENTS.md 전문을 매 세션 로드한다(토큰 절감 없음 — 공식 문서). 변경 이력 분리로 일부 상쇄하고, 세션 내에선 프롬프트 캐시 프리픽스에 앉아 턴당 한계비용은 낮다(캐시 동작은 문서 미확인). 남는 비용은 캐시 프리픽스 증가 + 긴 문서의 주목도 희석 — 후자는 CLAUDE.md 상단 앵커로 방어한다.
- **대안 기각**: `.claude/rules/`(경로 스코프 조건부 로딩)는 "특정 파일 작업 시에만" 로드라 "매 사용자 출력마다" 적용해야 하는 출력 언어 규칙에 안 맞는다. 앵커만 두고 임포트 안 하기(Y안)는 규칙마다 수작업 앵커가 필요하고 누수의 일반 해결이 아니라 기각.

## 검증

- **임포트 승인**: `@AGENTS.md`는 프로젝트 내부 경로라 외부 임포트 승인 다이얼로그 대상이 아니다(같은 저장소). 순환 없음(AGENTS.md는 CLAUDE.md를 임포트하지 않음).
- **단일 원본**: 전문 규칙은 AGENTS.md, CLAUDE.md는 요약 앵커만 — §11 로딩 규율로 재비대 차단.
- **Codex**: AGENTS.md 네이티브 로드라 무영향. 변경 이력 분리도 규칙이 아닌 로그라 Codex 규칙 로딩에 무영향.
