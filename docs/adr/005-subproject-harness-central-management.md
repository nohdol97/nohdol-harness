# ADR 005 — 하위 프로젝트 하네스 중앙 관리 (원본 루트 추적 + 심링크 배포)

- **날짜**: 2026-07-12
- **변경 내용**: 하위 프로젝트 하네스의 원본을 루트 `.agents/projects/<이름>/`에 두고 루트 저장소가 추적하며, `project/<이름>/`에는 심링크로 배포하고 하위 git 저장소에는 `.git/info/exclude`로 로컬 제외한다. 하위 전용 스킬·에이전트는 루트 `.agents/`에 `<프로젝트>-` 접두어로 생성하고, 하위 전용 훅은 루트 settings에 경로 분기형으로만 둔다.
- **대상**: AGENTS.md 12절(신설), metaskill SKILL.md(생성물 요건·체크리스트), orchestrate SKILL.md(공통 팀원 규칙 ①, B모드 프롬프트 규약), README, `.agents/projects/`(신설)
- **사유**: 사용자 확정(2026-07-12) — 스킬·에이전트는 루트에서 단일 관리하고, 하위 프로젝트 git에는 하네스를 올리지 않는다. 단순 gitignore 안(사용자 원안)은 하네스가 어디에도 추적되지 않아 유실·이력 부재 결함이 있어, 루트 추적 + 심링크 배포로 보완했다.

## 배경 — cwd 로드 문제 (1(a))

Claude Code의 스킬·훅·권한 설정은 세션 시작 디렉토리 기준으로만 로드된다. 루트에서 시작한 세션(크로스 프로젝트 작업)은 하위 프로젝트의 `.claude/` 실행 계층을 로드하지 못하고, 그 역도 마찬가지다. 문서(AGENTS.md)는 라우팅 규칙이 읽기를 강제하므로 유일하게 cwd와 무관하게 작동하는 계층이다.

## 결정

| 결정 | 내용 | 근거 |
|---|---|---|
| 규칙은 문서 우선 | 가드레일·필수 규칙은 AGENTS.md에만 싣고, 스킬·훅은 편의 장치로 취급 | 문서는 cwd와 무관하게 작동하는 유일한 계층 |
| 원본 루트 추적 | 하위 하네스 원본은 `.agents/projects/<이름>/`(AGENTS.md·CLAUDE.md·adr/), 루트 저장소 추적 | gitignore만 하면 어디에도 추적되지 않아 유실. REGISTRY.md 선례(ADR 004) — 설치 환경별 데이터도 추적한다 |
| 심링크 배포 | `project/<이름>/`에 AGENTS.md·CLAUDE.md 심링크 + `.claude/agents`·`.claude/skills` → 루트 `.agents/` 심링크. 하위 전용 `.agents/` 없음 | 단일 프로젝트 세션도 표준 위치에서 하네스·스킬을 찾는다. 루트의 `.claude → .agents` 심링크 철학(11절)과 동일 |
| 하위 저장소 로컬 제외 | 하위 git 저장소가 있으면 심링크들을 `.git/info/exclude`에 등록 | 하위 저장소(회사·공유 저장소 포함) 파일을 일절 건드리지 않고 하네스를 git 밖에 둔다 |
| 스킬·에이전트 루트 단일 소스 | 하위 전용 스킬·에이전트는 루트 `.agents/`에 `<프로젝트>-` 접두어로 | 하위에 두면 루트 세션에서 로드되지 않음(cwd 문제). 접두어로 소속 식별 |
| 훅은 루트 경로 분기형 | 하위 전용 훅은 루트 `.claude/settings.json`에 두고 스크립트가 대상 경로를 검사 | 훅은 세션 루트 것만 실행됨 — 우회로가 없다 |
| 팀원 하네스 주입 | orchestrate 팀원·서브에이전트는 작업 전 `project/<이름>/AGENTS.md` 읽기를 규칙화 | 서브에이전트도 하위 훅·설정을 상속받지 못함 |
| 트레이드오프 수용 | 하위 저장소만 단독 클론한 환경(원격 세션 등)에는 하네스 없음 | 하위 프로젝트 작업은 이 워크스페이스 안에서 수행하는 것을 전제로 한다 |
