# 신규 프로젝트 스캐폴딩 절차

"프로젝트 새로 만들어줘" 요청 시 이 절차를 따른다. 하네스 없는 프로젝트를 만들지 않는다 — 프로젝트 생성과 하네스 생성은 한 몸이다.

## 1. 구체화 인터뷰 (필수)

최소 확인 항목:

| 항목 | 왜 필요한가 |
|---|---|
| 스택 (언어·프레임워크·패키지 매니저) | 스캐폴딩 도구와 초기 스킬 결정 |
| 배포 방식 (k8s? 서버리스? 정적?) | 연관 프로젝트(k8s, gitops)와 가드레일 범위 결정 |
| 연관 프로젝트 | 레지스트리 "연관 프로젝트" 컬럼 = 크로스 프로젝트 라우팅 근거 |
| 반복 작업 예상 | 초기 스킬/에이전트 후보 |

## 2. 디렉토리·기본 스캐폴딩

- `<프로젝트명>/` 디렉토리 생성. 스택 표준 스캐폴더 사용 (예: `npm create vite`, `uv init`). 스캐폴더가 없으면 최소 구조만 수동 생성.
- git: 루트 모노 워크스페이스 저장소를 그대로 쓴다. 프로젝트별 별도 저장소가 필요하면 사용자와 확정 후 ADR 기록.

## 3. 하네스 생성

```
<프로젝트명>/
├── AGENTS.md          # 첫 줄: "이 문서는 루트 AGENTS.md를 상속한다."
├── CLAUDE.md          # 포인터: ## 하네스: {도메인} + 목표 + 트리거 + 변경 이력 1행
├── .agents/
│   ├── agents/        # 이 프로젝트 전용 에이전트 (agent-rules.md 템플릿)
│   └── skills/        # 이 프로젝트 전용 스킬 (스킬 공통 규칙 4가지)
├── .claude/
│   ├── agents -> ../.agents/agents    # 심링크
│   └── skills -> ../.agents/skills    # 심링크
└── docs/adr/          # 구조적 결정 기록
```

- 하위 AGENTS.md에는 **도메인 특화 규칙만** 담는다. 공통 관심사(git, 문서, 가드레일, 라우팅)는 루트에 있고 상속된다 — 복사하지 않는다. 이유: 복사본은 루트 갱신 시 어긋난다.
- 심링크 생성: `ln -s ../.agents/agents .claude/agents && ln -s ../.agents/skills .claude/skills`
- 심링크 검증: `ls -la .claude/` 로 링크 대상 확인. 실패 환경이면 sync 스크립트 대체 + ADR 기록.

## 4. 루트 레지스트리 갱신 (누락 금지)

루트 AGENTS.md 프로젝트 레지스트리에 행 추가:

| 이름 | 경로 | 스택 | 역할 | 연관 프로젝트 | 하네스 유무 |
|---|---|---|---|---|---|
| web | `web/` | Next.js | 사용자 웹 | k8s, gitops | ✅ |

## 5. 마무리

- 루트 AGENTS.md 변경 이력에 1행 추가 (같은 커밋에 포함).
- metaskill 완료 판정 체크리스트로 자체 검증.
- 사용자에게 커밋 여부 확인 (자동 커밋 금지).
